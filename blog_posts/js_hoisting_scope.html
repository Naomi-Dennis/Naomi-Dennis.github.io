Scope

Scope is the environment variables live in. Depending on where the variable is defined, the scope will be able to see it.
There are two basic kinds of scope, let’s look at them below. Consider the following example:

Local

Local scope is defined within a block. This can be the block of a function decleration, function expression, for loops, while loops, etc. Local variables are defined within the block and can only be used within that block. In this case, the local variable is y.

Global

Global scope is defined outside of any block. Variables, function expressions, funcion declerations, etc. defined in the global scope can be used within any inner blocks. In this case, the global variable is x.

Stack

When your code is run, all actions are placed on something called a stack. Stack is a data structure that can be thought of as a stack of plates. If you needed a plate from a stack of plates, you would take the top plate off the stack and use it. This is how the browser interacts with your code. Let & Var

In JavaScript versions of yore, the var keyword was the only way to define a variable that wasn’t a class or a function. Variable is still used in JavaScript, but it’s being phased out because it messes with the execution context.

For instance, the x variable from the example is used before its defined. The let keyword is the updated version of var, introduced in ES6. The simplest definition of let is it can only be used in its scope. If it’s defined in the global scope, it can be used globally; if it’s defined in the local scope, it can only be used locally.

The Stack Re-Visited

So far we’ve covered how the stack is created and executed; defined the scope and the differences between the let and var keywords. Now, we bring all the topics together in revisiting how the stack is composed and executed.

General rules for hoisting:

  1. Add all variable names and function declerations to the lookup table.
    i. If the name is already there, don’t add it again.
  2. Execute the code from the top down.
    i. If a variable that’s already defined is defined again; change its value.

Consider the following example:

Create Lookup Table
  1. ignore console.log(num1) since it’s a function call
  2. Add num1 variable name to the lookup table
  3. Add num2 variable name to the lookup table
  4. ignore console.log(num2) since it’s a function call
Execute code
  1. Call console.log(num1)
  2. Define num1 as 2
  3. Define num2 as 5
  4. Call console.log(num2);

Conclusion

Now that we’ve learned so much about hoisting, let’s answer the questions asked at the start of the article.

  • Why is x undefined?
    • Its variable nams is added to the lookup table, but its referenced before its set to anything.
  • Why is plus defined?
    • plus is a function decleration, so it’s added to the lookup table, along with its block in prior to the execution.
  • Why is x undefined when I pass it to console.log, but it is defined when I call plus?
    • By the time the browser reaches this log call, x has been defined in the lookup tablle.
